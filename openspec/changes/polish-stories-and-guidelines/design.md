## 背景 (Context)

当前《小金故事集》包含 62 集已完成的 Markdown 格式故事。虽然内容完整，但缺乏统一的质量标准，存在标点符号误用、语句不通顺等问题，且部分内容可能偏离了微观视角设定。此外，项目希望通过故事传递“自我调节”和“自我认知”等核心教育价值观，但目前的指南和故事内容并未明确体现这一点。

我们需要设计一套流程，既能高效地批量润色现有故事，又能确保未来的创作符合新的质量标准和价值观要求。

## 目标与非目标 (Goals / Non-Goals)

**目标：**
*   制定并执行一套标准化的润色流程，覆盖所有 62 集故事。
*   将新的质量标准（标点、适龄性、微观逻辑）和教育价值观（自我调节、自我认知）落实到 `GEMINI.md` 和 `conductor/product-guidelines.md` 中。
*   通过润色提升故事的可读性和教育价值，使其更适合 7-12 岁儿童。
*   确保所有故事文件的格式和元数据保持一致。

**非目标：**
*   重写故事的核心情节或改变角色设定（除非严重违背微观逻辑）。
*   新增或删除现有的集数。
*   对 Python 分析工具进行大规模重构（仅做必要的配置调整以支持新规则）。

## 决策 (Decisions)

### 决策 1：分阶段批量润色策略
**方案**：不一次性处理所有文件，而是将 62 集故事按“季”或“批次”分组（例如每 10-15 集一批）进行处理。
**理由**：
*   **质量控制**：分批处理允许我们在每批结束后进行审查，及时发现并纠正系统性错误，避免错误扩散到所有文件。
*   **上下文管理**：AI 在处理大量文件时可能会丢失上下文或产生幻觉。分批处理能保持上下文清晰。
*   **进度可视**：更容易追踪进度和回滚。

### 决策 2：先更新指南，后润色故事
**方案**：首先修改 `GEMINI.md` 和 `conductor/product-guidelines.md`，明确新的标准。只有在指南更新并确认无误后，才开始对故事文件进行润色。
**理由**：指南是润色工作的“法典”。AI Agent 需要依据最新的指南来执行润色任务。如果指南未更新，润色工作将缺乏依据或标准不一。

### 决策 3：人工辅助的自动化检查
**方案**：利用现有的 Python 脚本（或简单的 shell 命令）进行基础的格式检查（如标点符号扫描），然后由 AI Agent 进行深度的语义润色。
**理由**：
*   **效率**：机器擅长发现格式错误（如省略号不是6个点）。
*   **质量**：AI 擅长理解语义、语气和价值观植入。
*   **结合**：两者结合能最大化效率和质量。

## 风险与权衡 (Risks / Trade-offs)

*   **风险**：润色过程中可能意外改变故事原意或破坏伏笔。
    *   **缓解**：严格遵守“最小化修改原则”。在 Prompt 中明确指示：仅修正表达，不改动情节。分批提交 Git，确保可以轻松回滚。
*   **风险**：过度强调教育意义导致故事变得说教。
    *   **缓解**：在指南中强调“隐性教育”和“通过后果体现价值观”，并在润色示例中提供正反例对比。
*   **风险**：62 个文件的工作量较大，可能导致任务执行时间过长。
    *   **缓解**：使用脚本辅助，并接受分批次完成（不需要在一个会话中完成所有工作）。

## 迁移计划 (Migration Plan)

1.  **指南更新**：
    *   修改 `GEMINI.md`。
    *   修改 `conductor/product-guidelines.md`。
2.  **试运行 (Pilot)**：
    *   选取 1-2 集典型故事（如第1集和第10集）进行润色。
    *   人工审查润色结果，微调 Prompt 和指南。
3.  **批量执行**：
    *   按顺序对剩余故事进行批量润色（建议每 10 集一个批次）。
    *   每批次完成后提交 Git。
4.  **最终验证**：
    *   运行现有的内容分析工具 `scripts/analysis/run_scan.py` 确保无一致性错误。
